## 🟢 CONFIRMED (можно строить)

### Платформа и подход
- C1. Платформа сайта — **WordPress** (голос #15). [ТЗ 3, 8.1]
- C2. Встраивание делает **исполнитель**, программиста клиента не привлекают (#16). [ТЗ 7]
- C3. Стек/библиотеки — **на усмотрение исполнителя**, главное быстро и без зависаний (#17). [ТЗ 7]

- C6. Undo / удалить / переместить элементы между зонами (#19). [ТЗ 7]

### Конструктор формы
- C7. Палитра цветов → по клику карусель **готовых фото** форм этого цвета (НЕ программная перекраска) (#1). [ТЗ 2.1]
- C8. **Перёд + спина одновременно**, спина приоритетна (~95% заказов) (#2). [ТЗ 2.2]
- C9. Гетры на макете убрать, форма без шорт допустима (#2). [ТЗ 2.2]
- C10. **Плечо НЕ на теле** — отдельной картинкой сбоку (клик/наведение → загрузка/надпись) (9.3). [ТЗ 2.2, 9.3]
- C11. **Зоны фиксированные**; клиент начертит зоны по размерам на макете (#3, 9.1). [ТЗ 2.3, 9.1]
- C12. Зоны: спина фамилия+номер, грудь лого мал./надпись, спина лого, плечо лого, шорты номер+лого (включено). [ТЗ 2.3]
- C13. Текст: два поля **фамилия + номер**; поп-ап выбора шрифта; по умолчанию **РПЛ** (#9). [ТЗ 2.4]
- C14. **Лимита символов нет** (можно 2 строки) (#9). [ТЗ 2.4]
- C15. Выбор цвета шрифта (#9). [ТЗ 2.4]
- C16. **Дуга (изгиб) — НЕ делаем** (batch-4). [ТЗ 11.3]
- C17. **Кириллические шрифты добавить** (клубные — латиница); РПЛ поддерживает RU+латиницу (batch-4, #9). [ТЗ 11.2, 10.3]
- C18. Текст авто-подгоняется под зону (1 цифра = высота зоны; 2+ упирается в ширину, центр) (batch-2 #1). [ТЗ 9.2]
- C19. Логотипы на форму **пользователь грузит сам** (#7). [ТЗ 2.5]

## 🟡 ASSUMED (требует подтверждения перед опорой в архитектуре)

- A1. **Fabric.js v6** как canvas-движок.
  - Почему: зрелая либа под интерактивный canvas, C3 даёт свободу выбора.
  - Риск: если не подойдёт по перфу на слабых мобилках — переписать слой рендера.
  - Валидация: перф-замер на реальном устройстве в фазе 1.
- A2. **Отдельный WP-плагин** (не тема, не внешний сервис) как контейнер.
  - Почему: нужны админка, REST, WooCommerce-хуки, media (C35, C33).
  - Риск: если у клиента жёсткие ограничения хостинга/темы — скорректировать.
  - Валидация: доступ к админке + проверка окружения WP.
- A3. **Зоны в долях (0..1) canvas**, а не пиксели/см.
  - Почему: пропорциональность на мобилке (C18, ТЗ 9.2).
  - Риск: низкий; конвертация см→доли один раз при разметке.
- A4. **A3 = 30×40 см** как верхний предел принта футболки.
  - Почему: клиент сам колебался «А2 или А3… 30 на 40» (#4).
  - Риск: если нужен A2 — поменять один параметр.
  - Валидация: подтвердить у клиента число.
- A5. **Хранилище конфига** — WP options (прайс/сетки) + CPT (формы/шрифты).
  - Риск: низкий; влияет на реализацию админки, не на UX.
- A6. **remove-bg порогом по белому** достаточно для типового кейса.
  - Почему: клиент описал «лого на белом фоне» (11.5).
  - Риск: на градиенте/не-белом — плохо; позиционировать «для белого фона».
  - Валидация: тест на реальных лого клиента.
- A7. **Снапшот заказа** (design JSON + зафиксированная цена + версия зон) сохраняется в заказ.
  - Почему: правки конфига админом не должны ломать старые заказы.
  - Риск: без него — рассинхрон; наше инженерное решение, подтверждения клиента не требует.
- A8. **Цена векторизации 150 ₽/шт, пакет 50 = 6000 ₽** — наше предложение.
  - Риск: клиент может торговаться; влияет на деньги, не на код.
  - Валидация: ответ клиента на сообщение-4.
