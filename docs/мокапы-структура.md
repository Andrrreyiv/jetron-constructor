## ⚠️ Проблема: мокапы НЕ стандартизированы

Это подтверждает риск №2 из ТЗ. Разнобой:
- **Разные форматы**: JPG, PNG (в т.ч. с прозрачным фоном «фон»), WEBP.
- **Разный набор видов**: у одних есть перёд+спина, у других один вид.
- **Разный набор цветов** у каждой линейки (у Champion нет чёрного, у Star нет жёлтого и т.д.).
- **Разная геометрия/ракурс** у каждой модели — зоны нанесения из макета Champion не подойдут другим один-в-один.

### Что это значит для работы
- Зоны нанесения нужно задавать **отдельно под каждую модель** (координаты в пикселях на конкретном изображении). 8 линеек × перёд/спина = десятки конфигураций зон.
- Либо клиент выравнивает все фото под единый шаблон (как он сам предлагал «прогнать через ИИ»), тогда зоны задаются один раз.
- Это ключевой разговор по объёму: ручная разметка зон под каждую модель — существенная работа сверх «двух конструкторов».

### Логика «выбрал цвет → карусель форм»
Из мокапов видно: при выборе цвета (напр. синий) показываются все модели этого цвета:
- синий есть у: Champion, Space (бирюзовый?), Star, Venom, Winner, Legend (синяя), New (нет) ...
Нужна карта модель × цвет × вид, чтобы фильтровать карусель по выбранному цвету.
