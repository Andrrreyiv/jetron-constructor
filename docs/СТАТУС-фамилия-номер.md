# Статус: нанесение «Фамилия + Номер» (отложено)

Дата: 2026-07-15. Решение клиента: **временно убрать фамилию/номер с макета** — авто-размещение
на композитных мокапах выглядит криво. Весь остальной функционал конструктора работает.
Это заморозка, **не отмена**: по ТЗ (§3, C13–C18) фамилия+номер — часть итоговой поставки.

## Что сделано (сохранено в коде/конфиге)
- Зоны **остались** в `src/config/mock-config.json`:
  - `zoneTemplate`: `name` (спина, цена 600), `back_number` (спина, 0), `chest_number` (грудь, 300),
    `shorts_number` (шорты, 0).
  - `zoneSets.singleFront`: те же зоны для форм с раздельными картинками перёд/спина.
- Механика нанесения текста жива: `CanvasView.placeText` + авто-фит шрифта (`core/ZoneManager.fitFontSize`,
  покрыт тестами). Ничего из движка не удалено.
- Композитный рендер (перёд+спина+гетры на одном холсте) реализован в `ui/app.browser.js`
  (`viewList`/`_isComposite`/`targetView`/`renderAll`) — фамилия/номер спины технически МОГУТ рисоваться,
  как только вернём опции и выверим координаты.

## Что сейчас отключает фамилию/номер
- Из `placementOptions` (в `mock-config.json`) **удалены** опции `name_number` и `chest_number`.
  → в панели их нет, на макете рамок нет (рамка показывается только у зон с доступной опцией — фильтр в `renderAll`).
- Зоны как данные сохранены — при возврате опций всё поднимется.

## Почему криво (для будущей доработки)
- Композитные мокапы (31 форма, `front===back`) — одна картинка перёд+спина+гетры рядом; координаты
  спины выверены под правую половину. Текст спины садится, но по вертикали/дуге ещё грубо.
- Формы `singleFront` (12) имеют «back»-картинку, которая **тоже композит** (перёд+спина+шорты),
  а координаты `zoneSets.singleFront` для спины центрированы (~x=0.5) → попадают в зазор между рубашками.
  Нужны отдельные координаты для этих картинок ЛИБО перевод их в единый композитный холст.

## Как вернуть (чек-лист)
1. Вернуть в `placementOptions`:
   - `{ id:"name_number", kind:"name_number", nameZone:"back:name", numberZone:"back:back_number" }`
   - `{ id:"chest_number", kind:"number", zone:"front:chest_number" }`
2. Выверить `box`-координаты `name`/`back_number` под каждый тип мокапа (композит vs singleFront);
   для singleFront — либо свои координаты, либо привести к единому композиту.
3. Рассмотреть дугу фамилии — по ТЗ дуга НЕ делается (C16), фамилия прямая по центру.
4. Проверка на живом сайте: 1 символ = высота зоны, 2+ ужимается по ширине (уже в тестах ZoneManager).
