# MVP-SCOPE.md — Jetron Constructor

Дата: 2026-07-09. Источник: `ТЗ.md`, `архитектура.md`, `REQUIREMENTS.md`.
Метод: `mvp-scope-cutter`. Правило: **это фазирование поставки, а не урезание ТЗ**. Финальная сдача = полный объём ТЗ. «v1» = что показываем клиенту первым и с чего собираем деньги/обратную связь.

> ⚠️ Важно: ТЗ — платный заказ с зафиксированным объёмом. Мы не «выкидываем» пункты, мы решаем **порядок** и **что можно отдать позже без потери ценности**. Всё, что помечено v2, всё равно входит в итоговую поставку.

---

## Core Value Proposition
Покупатель сам собирает футбольную форму (перёд+спина, фамилия+номер, логотипы по зонам), **видит live-цену** и оформляет заказ — без дизайнера и без «зависаний» на телефоне.

## Primary User Goal (v1)
Зайти с телефона → выбрать модель/цвет → нанести фамилию+номер и лого в зоны → увидеть стоимость → «в корзину»/заявку с макетом и файлами.

---

## v1 MUST HAVE (первый релиз — конструктор ФОРМЫ)

Обоснование выбора формы, а не футболки: **C4 — клиент явно расставил приоритет** (форма первая, она сложнее). Футболка проще и переиспользует то же ядро.

### 1. Ядро конструктора (`core/`)
- CanvasManager (Fabric.js v6), ZoneManager (зоны в долях 0..1), DesignState.
- Почему essential: без канвы и зон нет продукта.
- ТЗ: 2.2, 2.3, 9.1-9.2. Риск: A1 (перф на мобилке) — замер на этой фазе.

### 2. Перёд + спина одновременно, плечо сбоку
- Почему essential: 95% заказов идут по спине (C8); плечо — отдельной картинкой (C10).
- ТЗ: 2.2, 9.3.

### 3. Текст: фамилия + номер, авто-подгон под зону
- Два поля, поп-ап шрифта, РПЛ по умолчанию, выбор цвета, без лимита символов, дуга НЕ делается.
- Авто-фит: 1 цифра = высота зоны; 2+ упирается в ширину, центр.
- Почему essential: ядро персонализации формы.
- ТЗ: 2.4, 9.2, C13-C18.

### 4. Кириллические шрифты + 17 предоставленных
- Почему essential: без кириллицы русская фамилия = пустые квадраты (C17).
- ТЗ: 10.3, 11.2.

### 5. Загрузка логотипов пользователем + размещение в зонах
- Клиент грузит свои лого (до ~15 МБ, сжатие в web), без проверки качества.
- Почему essential: зоны груди/спины/плеча/шорт бессмысленны без загрузки.
- ТЗ: 2.5, 4, C19, C24-C25.

### 6. Палитра цветов → карусель готовых фото
- Клик по цвету → карусель реальных фото форм этого цвета (НЕ программная перекраска).
- Почему essential: так клиент выбирает модель (C7).
- ТЗ: 2.1.

### 7. Live-пересчёт цены + строка «Общая стоимость нанесения»
- Сумма позиций (фамилия+номер, лого, номера) + форма; базового сбора нет; скидки Jetron −5%/−5%.
- Почему essential: клиент отдельно требовал видимую цену вверху (C30).
- ТЗ: 5, C26-C30. **Безопасность: финальная цена валидируется на сервере (не доверять фронту).**

### 8. Undo / удалить / переместить между зонами
- Почему essential: C6 явно в ТЗ, базовый UX редактора.
- ТЗ: 7.

### 9. Мобильная версия
- Почему essential: обязательна (C5), большинство трафика.
- ТЗ: 7.

### 10. Оформление заказа
- Детское/взрослое → таблица размеров (из конфига) → предложить гетры → итог → «в корзину».
- В заказ уходят: все файлы + надписи + PNG-макет; письмо на grc2@bk.ru.
- Кнопка «Скачать изображение» (превью).
- **Снапшот заказа** (design JSON + зафиксированная цена + версия зон) — A7.
- Почему essential: без заказа конструктор не монетизируется (C31-C34).
- ТЗ: 6, 10.1.
- 🚫 **Зависит от U1 (WooCommerce да/нет).** Если не снят — v1 стартует в режиме «заявка на почту», WooCommerce-ветка включается по подтверждению.

### 11. Админка — минимум
- Менять цены, добавлять шрифты, редактировать размерные сетки (без кода).
- Почему essential: цены/линейки «постоянно меняются» (C35-C36), хардкод недопустим.
- ТЗ: 10.5, 10.1.

**Блокеры старта v1-формы:** U2 (топ-3 модели), U3 (координаты зон). До них — **фаза 0** (каркас на mock-данных) уже разблокирована.

## v2 (входит в поставку, отдаём после v1)

### A. Конструктор ФУТБОЛКИ
- Плотность 120/150/260/300, свободное размещение, предел A3, библиотека картинок по темам.
- Почему позже: клиент сам поставил её второй (C4); переиспользует ядро v1 → быстро.
- Work-around: нет — просто следующая фаза. ТЗ: 3, C20-C23.
- Зависит от A4 (подтвердить A3=30×40).

### B. Zone Mapper (админ рисует зоны мышью)
- Почему позже: на v1 зоны заводим по присланным клиентом координатам (U3) вручную/через JSON. Полноценный визуальный маппер — самый тяжёлый админ-компонент.
- Work-around: ручная разметка координат для топ-3 моделей.
- ТЗ: 2.3, 9.1. Зависит от U10 (в рамках цены или доплата — коммерческий блокер).

### C. Админка — расширенная
- Добавление новых форм через UI, управление каруселями фото, полный CRUD зон.
- Почему позже: v1-админки (цены/шрифты/сетки) хватает для запуска.
- ТЗ: 10.5.

### D. Кнопка «удалить фон» (remove-bg по белому)
- Почему позже: опция, ждём да/нет клиента (U6); порог по белому (A6).
- Work-around: клиент грузит лого уже без фона. ТЗ: 11.5.

### E. Векторизация как встроенная услуга-поток
- Почему позже: это офлайн-услуга исполнителя (vtracer/potrace), не блокирует конструктор.
- ТЗ: 11.4, C40.

## OUT OF SCOPE (не планируется в этой поставке)

### Программная перекраска форм
- Почему cut: клиент прямо отверг — только готовые фото (C7).

### Проверка качества загружаемых изображений
- Почему cut: клиент прямо сказал не делать (C25).

### Дуга/изгиб фамилии
- Почему cut: отменено клиентом 08.07 (C16).

### Печатный файл высокого разрешения
- Почему cut: нужен только превью-макет (C34).

### Платёжный провайдер / checkout
- Почему cut: это WooCommerce-checkout клиента, вне нашего объёма (U11). Наш выход = «в корзину»/заявка.

### Командные скидки сверх Jetron-%
- Почему cut из v1: не блокер, частично решено (C29); при запросе — в бэклог (U7).

## Roadmap

**Фаза 0 (сейчас, без блокеров):** каркас стенда на mock-конфиге — ядро, зоны, текст, цена на фейковых данных. Реальные данные потом заменят mock.

**v1 (MVP — конструктор формы):** после U2+U3 (+U1 для ветки заказа)
- Ядро + перёд/спина/плечо
- Текст + кириллица + авто-фит
- Загрузка лого + зоны
- Палитра→карусель
- Live-цена
- Undo/move, мобилка
- Заказ (WooCommerce или почта)
- Админка-минимум (цены/шрифты/сетки)

**v2 (после v1 + фидбек):**
- Конструктор футболки
- Zone Mapper
- Админка расширенная
- remove-bg (если да)
- Векторизация-поток

## Итог
v1 = полноценный **конструктор футбольной формы с заказом**. Это не урезание ТЗ, а его первый приоритетный блок (C4). Футболка, Zone Mapper и остальное — следующие фазы той же поставки. Единственное, что действительно не делаем никогда — то, что клиент прямо отверг (перекраска, проверка качества, дуга, печатный HD).
