# Хендофф: правки по комментариям клиента (2026-07-22)

Документ для другого аккаунта Claude, чтобы продолжить без потери контекста.
Проект: jetron-constructor (конструктор футбольной формы, vanilla JS + Fabric.js v6).
Деплой: jetronsport.ru (WooCommerce) + демо на GitHub Pages. Клиент: MacBook/Safari + телефон.

## Как запустить и проверять
- Локальный сервер: `npx --yes serve -l 8777 .` (конфиг для Preview MCP: `.claude/launch.json`, имя `jetron`).
- Тесты: `node --test` (30 шт, чистая логика в `src/js/core/`).
- Быстрая проверка синтаксиса: `node --check src/js/ui/<file>.js`.
- Админ-режим редактора зон: URL `?zones=edit` (для покупателя это no-op).
- Проверять и на ПК, и на мобильном (Preview MCP: `preview_resize` preset mobile 375px). Fabric-скриншот иногда таймаутит, надёжнее `preview_eval` по DOM.

## Статус 10 правок клиента

СДЕЛАНО и проверено вживую:
- **1. Монограмма «JS» (фирменный знак Jetron)** — ПОДКЛЮЧЕНА. Файлы `assets/jetron-monogram.png`
  (чёрная) и `assets/jetron-monogram-white.png` (белая), ссылки в `mock-config.json` →
  `branding.logo` / `branding.logoInverse`. Знак штатный (не опция, не скидка): ВСЕГДА на груди
  справа и на шортах слева, на спине нет. `app.browser.js`: `renderJetron()` зовёт `_placeBrand()`
  для `chest_logo_large` и `shorts_number`. Авто-контраст: `bgLuminanceAt(box)` меряет яркость ткани
  под боксом → тёмная ткань даёт белую монограмму, светлая — чёрную (иначе «JS» на чёрных шортах
  сливается). Бокс увеличен через `_brandBox` (w 0.135, h 0.05, центр по зоне-якорю).
  Для singleFront-форм (space-black/new-white) в `zoneSets.singleFront` координаты шорт подняты
  на `y:0.77` (было 0.58 — попадало на подол футболки). Проверено на champion/space-black/new-white.
- **4. Плашка линейки слева сверху + переход в каталог** — СДЕЛАНО. `app.browser.js`:
  `_renderLineBadge()` рисует `button.line-badge` абсолютом в левом верхнем углу `#stage`,
  текст = `${form.line} →` (Champion/Legend/…), обновляется при смене формы. Клик → `_goToLineCatalog()`
  → URL из `config.catalog` (`base + slug + suffix`, слаг из `catalog.lineSlugs[line]` или `line.toLowerCase()`),
  навигация через верхнее окно как у крестика выхода. ⚠️ **Слаги категорий — ПРЕДПОЛОЖЕНИЕ**
  (`/product-category/champion/` и т.д.), клиент должен подтвердить реальные URL категорий на jetronsport.ru.
- **Доп. правка «мокапы подтянуть к верху»** — решена автоматически правкой 4. Раньше плашка была
  блоком в потоке (`flex-basis:100%; margin-bottom`) и толкала макет вниз на ~36px. Сделав её
  абсолютным оверлеем, макет поднялся к верху сцены (холст на 55px от верха, было бы ~91px).
- **2, 3. Редактор зон** — контент зоны (номер/надпись/лого) едет и тянется ВЖИВУЮ вместе с рамкой,
  пока админ её двигает/масштабирует. Файл `src/js/ui/zone-editor.browser.js`: слушатели
  `object:moving`/`object:scaling` + метод `_syncContent()` (репозиция центра, обновление clipPath,
  перешрифтовка текста через `fitFontSize` или пересчёт масштаба картинки). `onModified` тоже зовёт `_syncContent`.
- **5. Серое поле снизу макета убрано** — причина: высота холста была жёстко 3:4 (под Champion 900x1200),
  а мокапы разного формата (Venom 800x728, Legend 790x649, Star 1080x1080 и т.д.) после вписывания по
  ширине оказывались ниже холста. Фикс в `src/js/ui/canvas.browser.js`, метод `setBackground`: после
  вычисления `scale` высота холста подгоняется под вписанную картинку (`this.canvas.setDimensions({height: img.height*scale})`).
  У Champion (0.75) ничего не меняется. Проверено на 8 формах: `grayBottomPx = 0`, ПК и мобильный.
  ВАЖНЫЙ нюанс: этот фикс делает дробные координаты общего `zoneTemplate` одинаковыми для всех
  пропорций (раньше на широких формах зоны были сжаты кверху). Если админ раньше ВРУЧНУЮ тюнил зоны
  через `?zones=edit` на широких формах под старую высоту холста — те сохранённые правки в `zones.json`
  могут чуть съехать. В демо/шаблоне ручных правок нет.
- **6. Шрифт крупнее** у «Комплектация»/«Детализация» (CSS, прошлая сессия).
- **7. «Детализация» чёрным** цветом (CSS, прошлая сессия).
- **8. Полоса над «Итого» убрана**, блок приподнят (CSS, прошлая сессия).
- **9. Рамка выбора модели** больше не режется слева (CSS `.model-carousel`, прошлая сессия).
- **10. Кнопка «Удалить фон»** (волшебная палочка) во всех 4 блоках загрузки логотипа.
  `src/js/ui/app.browser.js`: `WAND_SVG`, разметка `.opt-upload-row` с кнопкой `data-act="rmbg"`,
  обработчик клика + метод `removeBackground(src)` (dependency-free flood-fill от краёв холста,
  фон = среднее 4 угловых пикселей, tol=42, BFS помечает связный внешний фон прозрачным, выход PNG с alpha).
  CSS в `src/css/stand.css` (`.opt-upload-row`, `.opt-rmbg`): кнопки в один ряд, upload `flex:1 1 140px; min-width:0`,
  rmbg `flex:0 0 auto; white-space:nowrap`. Проверено: side-by-side и на ПК, и на 375px.

## Стандарты работы (не нарушать)
- Верификация: «запустил → увидел → работает», не «должно работать».
- TDD, править причину а не симптом, хирургические минимальные правки.
- Клиентский русский текст: живой язык, БЕЗ длинных тире.
- Файлы < 500 строк. После правок кода ВСЕГДА гонять тесты.
- Коммитить ТОЛЬКО по явной просьбе.

## Ключевые файлы
- `src/js/ui/app.browser.js` — главный UI: рендер видов/зон, панель опций, композит, загрузка/удаление фона.
- `src/js/ui/canvas.browser.js` — обёртка Fabric: `setBackground` (фикс 5), `placeText`/`placeImage`.
- `src/js/ui/zone-editor.browser.js` — админ-редактор зон (`?zones=edit`), фиксы 2/3.
- `src/css/stand.css` — стили стенда (фиксы 6/7/8/9/10).
- `src/config/mock-config.json` — формы, зоны (`zoneTemplate`), опции, цены.
- `docs/СТАТУС-фамилия-номер.md` — отдельный статус по замороженной опции «Фамилия+Номер».
